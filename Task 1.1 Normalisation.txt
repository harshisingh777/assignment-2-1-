(a) Functional Dependencies
Composite Primary Key
(student_id, course_code)

advisor_name → advisor_email

course_code → course_name, instructor_name

instructor_name → instructor_email


Partial Dependencies

A partial dependency occurs when part of the composite key determines a non-key attribute.

From student_id
student_id
    → student_name
    → department
    → advisor_name
    → enrollment_year

These depend only on student_id, not on the entire key

From course_code
course_code
    → course_name
    → instructor_name

These depend only on course_code.
